Part 1: Question 2

  1. A branch is a copy of the main so each person can have their own branch to make edits, this allows the code on main to stay error free. It also prevents two people working on the same block of code and making changes to the same code. A master branch is parent of all the branchs that come after.
  2. A commit is a set of changes that are made to a branch of files and they are usually small batches. Commits are normally made to branches seperate from main, commits/merges to main branch have to be reviewed and checked by other poeple working on project to ensure no conflicts. Commit messages are important because they allow you to understand what changes were made and when.
  3. A pull request is someone proposing changes to the main branch. When you have your commits made to a branch and you want to pull it into the main, this is important to projects as it will allow group members to audit your code before changes are made to main.
  4. When a merge is initiated GitHub combines the changes from one branch to another. Most often it is a remote branch merging into the main branch.
